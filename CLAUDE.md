# CLAUDE.md — AI 投资分析助手

## 项目定位
这是一个**多公司、多语言**的报告分析系统。支持 A股/港股/美股的：
- 年报/季报（中文简繁体、英文、粤语白话文）
- 招股书、券商研报、行业报告、公告/新闻
- 用户将报告放入 `reports/<公司名>/` 后告知 AI 即可自动分析

## 核心工作流

### 报告分两类，处理方式完全不同：

**A 类：公司财报**（年报/季报/招股书/业绩会）
→ 使用 `reports/analysis_framework.md` 的完整框架
→ 输出格式：变化雷达 + 单季度拆解 + 板块分析 + 成本溯源 + 财务健康度 + 估值 + 投资判断
→ 保存为 `reports/<公司名>/<公司名>_<报告类型>_<日期>.md`

**B 类：研报/市场评论/行业报告/论文/新闻**
→ **不套用** analysis_framework.md 的财报模板
→ 直接提取文本，阅读全文，总结核心观点和论据
→ 输出格式：作者/机构观点摘要 + 核心逻辑链 + 关键数据 + 对投资的启示
→ 按需保存摘要或只口头汇报

### 当用户说"分析报告"或"读报告"时：
1. 扫描 `reports/` 下所有子目录（排除 `_archive` 和 `_templates`）
2. 找出尚未分析的报告文件（PDF/Word/Excel/图片）
3. **先运行 `python scripts/read_report.py <报告路径>` 提取文本并检测语言**
   - 该脚本自动识别：简体中文/繁体中文/英文/粤语白话文/日文/韩文
   - 输出 UTF-8 编码的 `.txt` 文件，确保后续处理不出编码错误
   - 同时打印报告页数、字体、语言类型
4. **判断报告类型**：是 A 类（公司财报）还是 B 类（研报/评论/行业/论文）
5. **A 类**：读取 `reports/analysis_framework.md`，按完整框架分析
   **B 类**：直接阅读全文，总结核心观点、逻辑链和关键数据
6. 将分析结果写入 `reports/<公司名>/<公司名>_<报告类型>_<日期>.md`
7. 汇总发现告知用户

### 报告类型自动识别（多语言）：

**中文报告（简/繁体，含粤语白话文）：**
- "年报"/"年報"/"年度"/"年度報告" → 年报分析（深度）
- "季报"/"季報"/"季度報告"/"Q1"/"Q2"/"Q3"/"半年"/"半年報" → 季报分析（中等）
- "招股"/"招股書"/"IPO" → 招股书分析
- "行业"/"行業"/"市场"/"市場"/"供需" → 行业报告分析
- "研报"/"研報"/"评级"/"目標價" → 券商研报分析

**英文报告：**
- "10-K"/"Annual Report"/"annual report" → 年报分析（深度）
- "10-Q"/"Quarterly Report"/"quarterly report"/"Q1"/"Q2"/"Q3" → 季报分析（中等）
- "Earnings Call"/"earnings transcript"/"earnings release" → 季报/业绩会
- "S-1"/"F-1"/"Prospectus"/"IPO" → 招股书分析
- "investor presentation"/"investor day" → 投资者交流
- "analyst report"/"target price"/"rating"/"upgrade"/"downgrade" → 券商研报

**不确定时**：先打开文件阅读前几页判断类型，再询问用户确认。

### 语言处理规则：
- 分析输出语言默认使用**中文**，术语会自动映射（如 Revenue→营收, Gross Margin→毛利率）
- 如果报告是英文但用户要求英文输出，使用英文分析
- 粤语白话文报告（如港交所部分文件）按标准中文财务术语处理
- 繁体中文报告按简中处理，保持术语一致

### 跨市场会计准则自动适配：
| 市场 | 默认准则 | 关键差异处理 |
|------|---------|-------------|
| A股 | CAS (中国企业会计准则) | 与IFRS趋同 |
| 港股 | IFRS / HKFRS | 与CAS 99%一致 |
| 美股 | US GAAP | 研发费用全部费用化、LIFO存货、商誉定性评估 |
| 其他 | IFRS | 按IFRS标准分析 |

分析时注意：US GAAP 公司（如 NVDA）的研发费用全部计入了费用，毛利率比 CAS/IFRS 公司偏低（研发计入 COGS vs 计入费用），需要标注。

### 分析哲学：变化优先，重原因轻结果

每份报告的分析必须遵循"**变化 → 原因 → 影响**"思维链：

```
业务变化（干什么变了）
  ↓ 驱动
经营变化（怎么干变了）
  ↓ 驱动
业绩变化（结果变了）
  ↓ 影响
投资判断（要不要行动）
```

每个重大变化必须回答三个问题：
1. **什么变了？**（方向+幅度）
2. **为什么变？**（外因/内因，一次性/持续性）——这是最重要的
3. **会怎样？**（对投资逻辑的影响）

### 变化分类标签（分析时必须标注）

| 标签 | 含义 | 例子 |
|------|------|------|
| ◉ 结构性拐点 | 业务结构或竞争格局的根本性改变 | 亏损业务扭亏、新市场突破、商业模式升级 |
| ◉ 加速 | 增长斜率显著提高 | 收入增速连续2季+提升 |
| ◉ 减速 | 增长斜率显著下降 | 收入增速连续2季+下降 |
| ◉ 反转 | 趋势由负转正或由正转负 | 扭亏/由盈转亏 |
| ◉ 利润率突变 | 毛利率/净利率大幅变化 | 单季毛利率变化>3pp |
| ◉ 现金流异常 | CFO与利润严重背离 | CFO/净利润<0.5持续2季 |
| ◉ 风险暴露 | 新的重大风险出现 | 大额减值、诉讼、监管 |
| ◉ 里程碑 | 达到标志性节点 | 营收突破整数关口 |

### 分析时必须做到：
- **变化雷达放第一页**：3分钟内能读完所有重大变化
- **追溯原因而非罗列结果**：不能只说"营收增长20%"，要说清楚是量增还是价增驱动，是外因（行业景气/原材料价格）还是内因（份额提升/产品升级/降本增效）
- **年报也要拆单季**：全年数据之下，按最近4-8个季度拆解趋势，识别加速/减速/拐点
- **区分总量和单季**：总量看全年或TTM，单季看边际变化方向和速率
- 提取核心财务数据并与上期对比（从 `data/financials/` 读取历史数据）
- 标注数据来源（报告第几页/哪个章节）
- 对异常波动给出预警
- 生成的分析文件保存到对应公司文件夹

### 历史数据管理：
- 分析完成后，将关键财务指标追加到 `data/financials/<公司名>_<代码>.json`
- 格式见 `data/financials/_template.json`
- 每年/每季度一条记录，包含营收、利润、毛利率、净利率、ROE、现金流等
- 不同会计准则的数据会在字段中标注（如 `accounting_standard: "US GAAP"`）

## 技术说明
- 使用 Python 读取 PDF（PyMuPDF/pdfplumber）、Word（python-docx）、Excel（openpyxl）
- PDF 中的表格优先用 pdfplumber 提取
- **备用方案**：`python scripts/pdf2txt.py <pdf路径> [-o 输出路径]`
  （当内置 PDF 读取因 API 限制报错时使用，本地提取文本不依赖 API）
- 图片型 PDF 用 OCR 处理或提示用户提供文字版
- 英文 PDF 亦可正常处理，无需额外配置
- 编码统一用 UTF-8

## PPT 制作工作流

用户需求：发来文档（Word/PDF/MD/图片），自动套用公司PPT模板生成演示文稿。

### PPT 模板
- 模板文件：`ppt/产融7月证券交流-mao.pptx`
- 尺寸：16:9 宽屏
- 含封面、章节页、内容页（文字/表格/图表）版式
- 固定页眉、logo、配色

### 工作流
1. **阅读理解**：无论什么格式的输入，提取内容结构
2. **列出大纲**：呈现 PPT 页面规划供用户确认
3. **确认后执行**：用户批准后再调用模板生成最终 PPT
